API Endpoints
GET /diseases
Returns all 8 diseases with their ICD-10 codes, categories, and associated biomarkers.
{ "count": 8, "diseases": [ { "name": "Type 2 Diabetes", "icd10": "E11", "biomarkers": ["HbA1c", ...] } ] }
GET /biomarkers
Returns all 27 unique biomarkers with units, reference ranges, and which diseases they map to.
{ "count": 27, "biomarkers": { "HbA1c": { "unit": "%", "reference": "<5.7" } } }
POST /analyse
Main endpoint. Send patient info + biomarker values, get ranked disease scores back.
{ "patient_name": "Jane Doe", "biomarkers": { "HbA1c": 7.8, "FPG": 145, "Troponin I": 0.02 } }
Scoring Logic
⬡ How a Disease Score is Computed
1
Each submitted biomarker is compared against clinical threshold bands (warn / high / critical)
2
Each band returns a component score (e.g. 0.45 for warn, 0.80 for high)
3
Scores are multiplied by their marker weight and summed, then divided by total weight used
4
A coverage penalty applies if fewer markers are submitted — prevents false high confidence
5
All diseases are ranked by final score; top result becomes primary diagnosis
◈ Risk Level Thresholds
Critical
≥ 80%
High
65 – 79%
Moderate
45 – 64%
Low
25 – 44%
Minimal
< 25%
Inverted markers (e.g. Amyloid-β42, eGFR, Hemoglobin) score higher when values are lower than reference — handled automatically by the engine.
